strixd has been deployed as Service {{ include "mailstrix.fullname" . }} (ClusterIP, port {{ .Values.service.port }}).

It is an INTERNAL scan backend — there is no Ingress and the Service is
ClusterIP by design. Point your rspamd mailstrix.lua plugin at:

    http://{{ include "mailstrix.fullname" . }}.{{ .Release.Namespace }}.svc.cluster.local:{{ .Values.service.port }}

1. Verify it is up:

   kubectl --namespace {{ .Release.Namespace }} port-forward svc/{{ include "mailstrix.fullname" . }} {{ .Values.service.port }}:{{ .Values.service.port }}
   curl -s http://127.0.0.1:{{ .Values.service.port }}/version
   curl -s http://127.0.0.1:{{ .Values.service.port }}/health

{{- if not (or .Values.token.value .Values.token.existingSecret) }}

2. !! NO TOKEN SET !!
   strixd will return HTTP 503 on every /scan until you set a shared token.
   Set one of:
     --set token.value=YOUR_SECRET
     --set token.existingSecret=my-secret --set token.secretKey=token
   The rspamd plugin MUST present the same value.
{{- else }}

2. Token is configured (strixd will accept /scan from clients presenting it).
{{- end }}
